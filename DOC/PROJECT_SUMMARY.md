# Project Summary: POST Application Challenge

You need to build a fully functional **Post Application** (Blog-like app) using **Supabase** for the backend services.

## Core Features to Build

### 1. Supabase Integration
- **Authentication**: Email/Password login and signup.
- **Database**: Store post data (Title, Content, Author, Date, Category).
- **Storage**: Store and retrieve images for posts.

### 2. Pages & UI
- **Signup Page**: Form for new users to create an account.
- **Login Page**: Form for existing users to sign in.
- **Feed Page (Home)**: 
  - A dashboard showing a list of posts from all users.
  - Each post should display an image, title, author, date, and a summary.
  - "Read More" functionality.
- **Post Management**:
  - **Create Post**: A form/modal to add a new post (Title, Image Upload, Content).
  - **Edit Post**: Ability to modify existing posts.
  - **Delete Post**: Ability to remove posts.

### 3. Design Requirements
- Clean, modern UI (as seen in the provided screenshots).
- Responsive layout for web.
- Interactive elements (modals for creating posts).

## Technical Roadmap
1. [ ] **Setup**: Initialize a folder and set up Supabase project.
2. [ ] **Auth**: Implement Signup and Login logic.
3. [ ] **Database Schema**: Create a `posts` table in Supabase.
4. [ ] **Storage Setup**: Create a bucket for post images.
5. [ ] **Feed Implementation**: Fetch and display posts on the home page.
6. [ ] **CRUD Operations**: Implement Create, Edit, and Delete functionality.
7. [ ] **Deployment**: Push to GitHub and host on a platform like Firebase or Vercel.
